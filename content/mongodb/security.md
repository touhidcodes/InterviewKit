# MongoDB Security (Interview Q&A)

## 1. Why is security important in MongoDB?
To protect sensitive data and prevent unauthorized access.

---

## 2. How do you enable authentication?
By configuring MongoDB with users and passwords.

---

## 3. What is authorization?
Controlling access using roles and permissions.

---

## 4. What is RBAC?
Role-Based Access Control for managing user permissions.

---

## 5. How do you secure network access?

- Bind MongoDB to localhost or private IP  
- Use firewalls  
- Restrict open ports  

---

## 6. What is TLS/SSL?
Encryption for data in transit.

---

## 7. How do you secure data at rest?
Use encryption features provided by MongoDB.

---

## 8. What are common security risks?

- Open database ports  
- Weak passwords  
- No authentication enabled  

---

## 9. What is MongoDB Atlas?
A cloud-managed MongoDB service with built-in security features.

---

## 10. What are best practices?

- Enable authentication  
- Use strong passwords  
- Regular backups  
- Monitor access logs  

---

## Summary

Proper security practices are essential to safeguard MongoDB databases from unauthorized access and data breaches.
