# MongoDB Schema Design (Interview Q&A)

## 1. What is schema design in MongoDB?
Designing how data is structured in collections and documents.

---

## 2. Does MongoDB have a fixed schema?
No, MongoDB is schema-less.

---

## 3. What is a flexible schema?
Documents in the same collection can have different structures.

---

## 4. What are schema design patterns?

- Embedded pattern  
- Reference pattern  
- Bucket pattern  

---

## 5. What is the bucket pattern?
Grouping related data into a single document for efficiency.

---

## 6. What is the outlier pattern?
Separating large or infrequent data into another collection.

---

## 7. What is schema validation?
Enforcing rules on document structure.

---

## 8. Why is schema design important?

- Improves performance  
- Reduces complexity  
- Ensures scalability  

---

## 9. What are common mistakes?

- Over-normalization  
- Deep nesting  
- Ignoring query patterns  

---

## Summary

Proper schema design is key to building efficient MongoDB applications.
